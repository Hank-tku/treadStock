# 交互行为规范 -- 股势 TrendStock
版本：v1.0 | 日期：2026-04-10 | 依据：docs/design-spec.md v1.0, docs/prd.md v1.0
状态：Phase B -- 已锁定（Autopilot 默认确认，用户已确认所有交互意图）

---

## 总则

### 交互原则

1. **即时反馈**：所有用户操作（点击、滑动、开关）在 100ms 内产生视觉变化，不等待异步操作完成。
2. **操作可逆优先**：删除/移除等破坏性操作不弹窗确认，但提供 Toast 反馈和明确的恢复路径（重新添加）。
3. **零阻塞加载**：网络请求不阻塞用户操作。已有缓存数据立即展示，异步刷新后静默更新。
4. **手势克制**：仅在左滑删除/右滑置顶两个场景使用滑动操作，其余场景使用点击。

### 触摸目标

所有可交互元素最小触摸区域 44x44px（WCAG 2.1 AA）。实际可点击区域可通过 `padding` 扩展至 44x44，视觉元素可以小于此值。

### 动效常量

| 级别 | 时长 | 缓动曲线 | Flutter 实现 |
|------|------|---------|-------------|
| instant | 100ms | easeOut | `Curves.easeOut` |
| fast | 200ms | easeInOut | `Curves.easeInOut` |
| normal | 300ms | easeInOut | `Curves.easeInOut` |

不使用弹簧动画（`Curves.elasticIn/Out`）。金融工具调性要求动效克制、精确。

### 搜索框表单字段规范（适用于所有搜索场景）

| 字段属性 | 规范 |
|---------|------|
| 字段类型 | 文本输入（股票代码或名称） |
| 必填（required） | 否（可选操作，不输入不影响页面展示） |
| placeholder | 「搜索股票代码或名称」 |
| 校验规则（validate） | 输入字符数 >= 1 时触发搜索；无最大长度限制；不校验格式（用户可输入任意字符） |
| 输入类型 | `TextInputType.text`（不使用数字键盘，因为用户可能输入中文股票名称） |
| 错误状态 | 无前端校验错误（搜索无结果时展示「未找到匹配股票」，不视为校验错误） |
| 自动完成 | 关闭（`autocorrect: false`，`enableSuggestions: false`） |

## 页面 1：推荐列表页（/recommend）

### 1.1 页面进入

| 项目 | 规范 |
|------|------|
| 触发 | 用户点击底部 Tab「推荐」 |
| 转场动画 | 无（Tab 切换无动画，内容即时替换） |
| 首次数据 | 先渲染 SQLite 缓存 -> 异步请求 API -> 成功则静默刷新 UI，失败则显示缓存 Banner |
| 后续进入 | Riverpod `keepAlive` Provider 保持状态，Tab 切换不重新加载 |

### 1.2 下拉刷新（Pull to Refresh）

| 项目 | 规范 |
|------|------|
| 手势 | 垂直下拉，超过 60px 触发刷新 |
| 视觉 | 原生 `RefreshIndicator`（Material 风格），品牌色 #1A6AFF |
| 刷新中 | 显示 Skeleton 加载占位（仅列表区域，标题栏保持） |
| 成功 | Skeleton 消失，数据更新，无 Toast |
| 失败 | 列表恢复显示，顶部出现黄色缓存 Banner |

### 1.3 列表项点击

| 项目 | 规范 |
|------|------|
| 手势 | `onTap`（非 `onLongPress`） |
| 按下态 | 背景 #F7F8FA，100ms easeOut 过渡 |
| 释放 | 背景恢复 #FFFFFF，100ms easeOut |
| 跳转 | `Navigator.push` 到 `/stock/:code` |
| 转场动画 | iOS: `CupertinoPageRoute` 右滑推入；Android: `MaterialPageRoute` 底部推入（`fadeIn + slideUp`） |

### 1.4 列表项长按

| 项目 | 规范 |
|------|------|
| 行为 | 无操作。长按不触发任何行为（避免与滑动操作冲突，且无分享/复制需求） |

### 1.5 评分 Badge 交互

| 项目 | 规范 |
|------|------|
| 行为 | 纯展示，不可交互。点击事件穿透到列表项 |
| 计算中态 | 背景色 #EEEEEE，文字「...」（三个点），无动画 |
| 数据不足态 | 背景色 #EEEEEE，文字「N/A」，副文字「数据不足」 |

### 1.6 波段低谷标签交互

| 项目 | 规范 |
|------|------|
| 行为 | 纯展示，不可交互。点击事件穿透到列表项 |
| 位置 | 紧跟评分 Badge 右侧，或股票名称下方（由设计稿最终确定） |

### 1.7 缓存 Banner 交互

| 项目 | 规范 |
|------|------|
| 显示条件 | 网络请求失败且有缓存数据时显示 |
| 关闭 | 用户点击右侧 X 按钮关闭，关闭后本次会话不再显示（直到下次刷新失败） |
| 关闭动画 | 高度从 32px 收缩到 0，200ms easeInOut |
| 文案 | 「数据更新失败，显示缓存数据」 |
| 下方附加 | 「数据更新于 14:30」（caption 11px #8C8C8C） |

### 1.8 分组标题（短线波段 / 中线波段）

| 项目 | 规范 |
|------|------|
| 行为 | 纯展示，不可交互 |
| 折叠 | 本期不支持分组折叠（分组始终展开） |

### 1.9 状态变体交互

#### (a) 加载中（Skeleton）

| 项目 | 规范 |
|------|------|
| 触发 | 首次打开、下拉刷新中 |
| 视觉 | 6-8 行 Skeleton 卡片，灰色 #F0F1F3，shimmer 动画 |
| 用户操作 | 列表不可滚动交互（Skeleton 占位期间用户可看到但不能点击列表项），Tab 切换仍可用 |
| 动画 | `ShaderMask` + `LinearGradient` 循环动画，周期 1500ms |

#### (b) 加载失败（有缓存）

| 项目 | 规范 |
|------|------|
| 触发 | API 请求失败且 SQLite 有缓存 |
| 视觉 | 列表展示缓存数据，顶部黄色 Banner |
| 用户操作 | 列表可正常点击跳转（数据是有效的，只是不是最新） |

#### (c) 加载失败（无缓存）

| 项目 | 规范 |
|------|------|
| 触发 | 首次打开且 API 失败且无本地缓存 |
| 视觉 | 空状态：信号断开图标（64x64 #BDBDBD）+ 标题「暂无推荐数据」+ 副标题「检查网络后下拉刷新」 |
| 用户操作 | 可下拉刷新。无独立刷新按钮（提示下拉操作） |

#### (d) 非交易日

| 项目 | 规范 |
|------|------|
| 触发 | 当前日期为周末或法定节假日 |
| 视觉 | 副标题改为「非交易日，显示最近交易日数据」，分组标题旁显示日期 |
| 用户操作 | 与正常状态完全一致。列表展示最近一个交易日的数据 |

## 页面 2：关注列表页（/watchlist）

### 2.1 页面进入

| 项目 | 规范 |
|------|------|
| 触发 | 用户点击底部 Tab「关注」 |
| 转场动画 | 无（Tab 切换无动画） |
| 数据 | SQLite 本地数据，无网络请求（关注列表是纯本地数据） |

### 2.2 搜索框交互

#### 2.2.1 聚焦

| 项目 | 规范 |
|------|------|
| 触发 | 用户点击搜索框 |
| 视觉变化 | 边框 #E0E0E0 -> #1A6AFF（200ms easeInOut），出现阴影 `0 1px 2px rgba(0,0,0,0.05)` |
| 系统键盘 | 弹出系统键盘（文本类型），键盘上方无额外 toolbar |
| 自动聚焦 | 空状态时搜索框自动获得焦点（`FocusNode.requestFocus()`） |

#### 2.2.2 输入中

| 项目 | 规范 |
|------|------|
| 触发 | 输入字符数 >= 1 |
| 防抖 | 300ms debounce（最后一次输入后 300ms 执行搜索） |
| 搜索范围 | SQLite 本地股票数据库（全市场股票表），LIKE 匹配代码或名称 |
| 最大结果 | 10 条 |
| 结果展示 | 搜索框下方展开结果列表，覆盖在关注列表上方 |

#### 2.2.3 搜索结果列表

| 项目 | 规范 |
|------|------|
| 布局 | 全宽（减去 32px 水平内边距），紧贴搜索框下方 |
| 背景 | #FFFFFF，无阴影（直接覆盖在列表上方） |
| 分隔 | 1px border #EEEEEE |
| 每条结果 | 左侧：股票名称（body-lg 15px）+ 代码（caption 11px #8C8C8C）；右侧：按钮 |

#### 2.2.4 添加关注按钮

| 项目 | 规范 |
|------|------|
| 默认态 | 「添加关注」文字按钮，颜色 #1A6AFF |
| 按下态 | 文字颜色加深 #1554D6，100ms |
| 已关注态 | 「已关注」灰色文字 #BDBDBD，按钮 disabled，不可点击 |
| 添加中（loading） | 按钮变为 disabled，文字替换为 `CircularProgressIndicator`（直径 16px，品牌色） |
| 全局禁用 | 一只股票添加请求进行中时，搜索结果中所有添加按钮均变为 disabled（防重复提交） |

#### 2.2.5 搜索无结果

| 项目 | 规范 |
|------|------|
| 视觉 | 结果列表区域显示「未找到匹配股票」（body 13px #8C8C8C），垂直居中 |
| 高度 | 与有结果时保持相同高度（约 200px），避免跳动 |

#### 2.2.6 清空搜索

| 项目 | 规范 |
|------|------|
| 触发 | 用户点击搜索框右侧 X 按钮 |
| 行为 | 清空输入文字，收起搜索结果列表，搜索框恢复默认态（灰色边框） |
| 动画 | 搜索结果列表向上收起，200ms easeInOut |

#### 2.2.7 失焦（取消搜索）

| 项目 | 规范 |
|------|------|
| 触发 | 用户点击搜索框外区域、按返回键、或 Tab 切换 |
| 行为 | 收起搜索结果列表，保留输入文字，搜索框恢复默认边框 |
| 动画 | 搜索结果列表向上收起，200ms easeInOut |

### 2.3 列表项点击

| 项目 | 规范 |
|------|------|
| 行为 | 与推荐列表一致 -- 按下态背景 #F7F8FA，跳转到 `/stock/:code` |
| 转场动画 | 同 1.3 规范 |
| 禁止点击 | 左滑/右滑操作展开期间，列表项点击被禁用（避免误触） |

### 2.4 左滑删除

| 项目 | 规范 |
|------|------|
| 手势 | 水平向左滑动（`GestureDetector` onHorizontalDragUpdate），滑动距离 > 80px 展开操作按钮 |
| 视觉 | 列表项向左偏移，露出红色背景（#E6432D），背景中央显示白色「移除」文字 |
| 操作按钮宽度 | 80px |
| 保持展开 | 用户松手后如果滑动距离 >= 80px，保持展开状态（不自动收回） |
| 自动收回 | 如果滑动距离 < 80px，列表项弹回原位，200ms easeInOut |
| 多项展开 | 同一时间只允许一项展开（新项展开时，已展开的项自动收回） |

#### 2.4.1 点击「移除」

| 项目 | 规范 |
|------|------|
| 触发 | 用户点击红色「移除」区域 |
| 即时反馈 | 列表项从列表中移除，动画 300ms easeInOut（高度收缩到 0 + 透明度变 0） |
| 数据删除 | 从 SQLite watchlist 表中删除该股票记录 |
| Toast | 「已移除 {股票名称}」，持续 2s，位于底部 Tab 栏上方 16px |
| 撤销 | 本期不支持撤销（遵循"操作可逆优先"原则，用户可通过搜索重新添加） |

### 2.5 右滑置顶

| 项目 | 规范 |
|------|------|
| 手势 | 水平向右滑动，滑动距离 > 80px 展开操作按钮 |
| 视觉 | 列表项向右偏移，露出蓝色背景（#1A6AFF），背景中央显示白色「置顶」文字 |
| 操作按钮宽度 | 80px |
| 保持/收回 | 同 2.4 规范 |

#### 2.5.1 点击「置顶」

| 项目 | 规范 |
|------|------|
| 触发 | 用户点击蓝色「置顶」区域 |
| 即时反馈 | 列表项收回到原位（200ms），然后整个列表重新排列，被置顶的项滑动到列表顶部 |
| 动画 | 使用 `AnimatedList` 或 `AnimatedSwitcher`，列表项重新排序动画 300ms easeInOut |
| 数据更新 | SQLite 中更新该股票的 `pinned` 字段为 true，排序权重置顶 |
| Toast | 「已置顶 {股票名称}」，持续 2s |
| 置顶项视觉 | 列表项左侧出现蓝色图钉图标（16x16 #1A6AFF） |

#### 2.5.2 已置顶项再次右滑

| 项目 | 规范 |
|------|------|
| 行为 | 已置顶的项右滑时，操作按钮文字变为「取消置顶」（灰色背景 #8C8C8C） |
| 点击效果 | 取消置顶，该项回到正常排序位置，Toast「已取消置顶 {股票名称}」 |

### 2.6 空状态交互

| 项目 | 规范 |
|------|------|
| 触发 | 关注列表为空（SQLite watchlist 表无记录） |
| 视觉 | 搜索图标（64x64 #BDBDBD）+ 标题「暂无关注的股票」+ 副标题「搜索添加你感兴趣的股票」 |
| 自动聚焦 | 搜索框自动获得焦点，引导用户立即搜索 |
| 用户操作 | 等待用户在搜索框输入并添加 |

### 2.7 下拉刷新

| 项目 | 规范 |
|------|------|
| 行为 | 关注列表为纯本地数据，下拉刷新触发评分和价格的异步更新（调用股票 API 获取最新价格） |
| 视觉 | 同推荐列表 `RefreshIndicator`，品牌色 |
| 刷新中 | 价格和涨跌幅区域显示 Skeleton，其他信息保持 |
| 成功 | 数据静默更新（价格、涨跌幅、评分可能变化） |

## 页面 3：股票详情页（/stock/:code）

### 3.1 页面进入

| 项目 | 规范 |
|------|------|
| 触发 | 从推荐列表或关注列表点击股票项 |
| 转场动画 | iOS: `CupertinoPageRoute`（右滑推入，左滑返回）；Android: `MaterialPageRoute`（底部推入，顶部返回） |
| 预加载数据 | 路由参数包含 `code` 和 `name`，头部信息（名称、代码）从参数直接填充，不等待 API |
| 并行加载 | 进入页面后并行请求：公司基本信息 API + 新闻 API + 评分计算 |

### 3.2 返回导航

| 项目 | 规范 |
|------|------|
| iOS 返回 | 页面顶部左箭头按钮点击，或从屏幕左边缘右滑返回（`CupertinoPageRoute` 支持） |
| Android 返回 | 页面顶部左箭头按钮点击，或系统返回键 |
| 转场动画 | 与进入动画反向（iOS: 从右向左滑出；Android: 向下滑出） |
| 状态保持 | 返回后列表保持之前的滚动位置（Riverpod Provider 未 dispose） |

### 3.3 预警开关（Switch）

| 项目 | 规范 |
|------|------|
| 组件 | Material `Switch`，品牌色 #1A6AFF（开启态），#EEEEEE（关闭态） |
| 位置 | 股票头部信息右侧 |
| 开启 | 用户拨动开关 -> ON 状态（200ms 颜色过渡） -> 检查通知权限 -> Toast「已开启 {股票名称} 下跌预警」 |
| 关闭 | 用户拨动开关 -> OFF 状态 -> SQLite 更新 alert_config -> Toast「已关闭 {股票名称} 下跌预警」 |
| 无通知权限 | 开启时如果用户未授予通知权限 -> 系统弹出权限请求弹窗 -> 用户拒绝 -> 开关自动回弹到 OFF -> Toast「需要通知权限才能接收预警」 |

### 3.4 新闻列表项点击

| 项目 | 规范 |
|------|------|
| 手势 | `onTap` |
| 按下态 | 背景 #F7F8FA，100ms easeOut |
| 跳转 | 使用 `url_launcher` 打开外部浏览器（新闻原文 URL） |
| 无 URL | 如果某条新闻缺少 URL，点击无响应（列表项视觉上不显示按下态） |

### 3.5 公司信息区域

| 项目 | 规范 |
|------|------|
| 行为 | 纯展示，不可交互 |
| 长按 | 无操作 |

### 3.6 评分 + 指标卡片区域

| 项目 | 规范 |
|------|------|
| 行为 | 纯展示，不可交互 |
| 点击 | 无操作（卡片不是可点击的） |
| 计算中态 | 评分卡片背景 #EEEEEE，文字「...」，200ms 渐入 |
| 数据不足态 | 评分卡片背景 #EEEEEE，文字「N/A」，下方小字「数据不足」 |

### 3.7 每日跟踪摘要

| 项目 | 规范 |
|------|------|
| 行为 | 纯展示，不可交互 |
| 位置 | 评分卡片下方，灰色背景卡片（#F7F8FA） |
| 非交易日 | 标注「数据截至 4月9日 收盘」（最近交易日日期） |

### 3.8 下拉刷新

| 项目 | 规范 |
|------|------|
| 行为 | 同推荐列表规范。刷新价格、新闻、评分 |
| 刷新中 | 所有数据区域显示 Skeleton（公司基本信息不重新加载，使用缓存） |
| 部分失败 | 新闻加载失败不影响其他区域，新闻区域显示「新闻加载失败，点击重试」 |

### 3.9 新闻区域重试

| 项目 | 规范 |
|------|------|
| 触发 | 新闻加载失败后，用户点击「点击重试」文字 |
| 视觉 | 「点击重试」文字颜色 #1A6AFF，按下态颜色 #1554D6 |
| 重试中 | 替换为 `CircularProgressIndicator`（直径 16px） |
| 重试成功 | 显示新闻列表，Skeleton 过渡 200ms |
| 重试失败 | 保持错误提示，Toast「加载失败，请稍后重试」 |

### 3.10 状态变体交互

#### (a) 加载中

| 项目 | 规范 |
|------|------|
| 触发 | 首次进入详情页 |
| 头部信息 | 立即显示（来自路由参数，无延迟） |
| 价格区 | Skeleton（display-lg 大小的灰色矩形） |
| 评分卡 | Skeleton（三列灰色矩形） |

## 外部 API 错误码映射（Error Map）

股势 TrendStock 为纯客户端架构，所有数据通过客户端直接调用公开金融 API 获取。
以下为外部 API 返回错误码时的客户端统一处理规范：

### HTTP 错误码处理

| HTTP 状态码 | 含义 | 客户端处理行为 | 用户可见提示 |
|------------|------|-------------|------------|
| 400 (Bad Request) | 请求参数错误 | 不重试，记录日志 | Toast「请求参数错误」 |
| 401 (Unauthorized) | API 认证失败 | 检查 API Token 是否过期，尝试刷新 Token | 若 Token 无效：Toast「数据服务认证失败」+ 离线模式 |
| 403 (Forbidden) | API 访问受限（IP/频率） | 切换备用数据源 | Toast「数据访问受限，请稍后重试」 |
| 404 (Not Found) | 股票代码不存在或新闻已删除 | 使用本地缓存；新闻 404 则移除该条目 | 无提示（静默处理） |
| 422 (Validation Error) | API 参数校验失败 | 不重试，记录日志 | Toast「数据请求格式错误」 |
| 429 (Rate Limit) | API 请求频率超限 | 暂停请求 60 秒后自动重试 | Toast「请求过于频繁，{N}秒后重试」 |
| 500 (Server Error) | 服务器内部错误 | 使用本地缓存，30 秒后自动重试 1 次 | 缓存 Banner「数据更新失败，显示缓存数据」 |
| 502/503/504 (Bad Gateway / Service Unavailable / Gateway Timeout) | 服务不可用 | 使用本地缓存，不自动重试（等待用户下拉刷新） | 缓存 Banner「数据服务暂不可用」 |

### 网络层错误处理

| 错误类型 | 触发条件 | 客户端处理行为 | 用户可见提示 |
|---------|---------|-------------|------------|
| SocketException / 网络断开 | 设备无网络连接 | 切换离线模式，所有数据使用 SQLite 缓存 | 离线 Banner「离线模式，数据更新于 {时间}」 |
| TimeoutException | 请求超过 10 秒未响应 | 取消请求，使用缓存数据 | 缓存 Banner「请求超时，显示缓存数据」 |
| HandshakeException | HTTPS 证书错误 | 使用缓存，记录安全日志 | Toast「网络安全连接失败」 |
| DioException (connection refused) | API 服务器拒绝连接 | 使用缓存，标记该 API 为不可用 | Toast「无法连接数据服务」 |

### 错误处理原则

1. **所有网络错误不展示全屏错误页**，统一使用内联 Banner 或 Toast 提示。
2. **有缓存数据时始终展示缓存**，即使数据已过期（标注更新时间即可）。
3. **自动重试仅限 1 次**，且间隔 >= 30 秒。避免无限重试消耗用户流量。
4. **429 频率限制特殊处理**：显示倒计时，告知用户何时可重试。

## 数据刷新策略

### 7.1 刷新时机

| 场景 | 触发 | 数据范围 |
|------|------|---------|
| 首次打开 App | App 启动 | 全部（推荐列表 + 价格） |
| Tab 切换 | 用户点击 Tab | 不刷新（使用缓存状态） |
| 下拉刷新 | 用户手动下拉 | 当前页面全部数据 |
| 进入详情页 | 从列表跳转 | 该股票全部数据 |
| 后台到前台 | App 从后台恢复 | 仅刷新价格（其他数据使用缓存） |
| 定时刷新 | workmanager 每 30 分钟 | 全部价格数据 |

### 7.2 缓存优先策略

```
用户打开页面
  -> 立即渲染 SQLite 缓存
  -> 异步请求 API
  -> 成功: 静默更新 UI（无 Toast、无动画跳变）
  -> 失败: 显示缓存 Banner（如果上次缓存时间 > 30 分钟）
  -> 无缓存且失败: 空状态 + 错误提示
```

### 7.3 数据过期判定

| 数据类型 | 过期时间 | 过期后行为 |
|---------|---------|----------|
| 推荐列表 | 交易日内 30 分钟 | 标注「数据更新于 {时间}」 |
| 关注列表价格 | 交易日内 5 分钟 | 标注「数据更新于 {时间}」 |
| 新闻 | 24 小时 | 不标注（新闻时效性由发布时间判断） |
| 评分 | 当日有效 | 非交易日显示最近交易日评分 |

## 疑问记录（Phase A 提出的问题）

以下问题在 Autopilot 模式下使用默认行为，无需人工确认：

| # | 问题 | 默认决策 | 理由 |
|---|------|---------|------|
| Q1 | 搜索结果点击是直接添加还是跳转详情？ | 直接添加到关注列表 | PRD Gherkin 明确「点击搜索结果中的某只股票 -> 该股票添加到关注列表」 |
| Q2 | 左滑删除是否需要二次确认弹窗？ | 不需要，直接执行 | 可逆操作（重新添加即可），PRD 未要求确认弹窗 |
| Q3 | 置顶操作是否需要二次确认？ | 不需要 | 完全可逆，可随时取消置顶 |
| Q4 | 搜索结果最多展示几条？ | 10 条 | PRD Gherkin 明确「最多 10 条」 |
| Q5 | 新闻点击跳转到外部还是 App 内 WebView？ | 外部浏览器（url_launcher） | 避免内嵌 WebView 的合规风险和 UX 问题 |
| Q6 | 是否支持列表项拖拽排序？ | 不支持 | 本期仅支持滑动操作（删除/置顶），拖拽排序 v2 考虑 |
| Q7 | 评分是否可手动修改？ | 不支持 | 系统自动计算，用户不可干预 |
| Q8 | 详情页是否支持下拉关闭（类似 BottomSheet）？ | 不支持 | 详情页是全屏页面，使用导航栏返回 |
| Q9 | 非交易日推荐列表是否显示提示？ | 是，副标题标注 | design-spec.md 已定义 |
| Q10 | Toast 是否支持滑动关闭？ | 仅失败消息支持 | 成功消息持续时间短（2s），不需要手动关闭 |
