## 页面架构

### 导航层级

```
App
├── Bottom Tab Bar
│   ├── 推荐 (RecommendedTab)    -- /recommend
│   └── 关注 (WatchlistTab)      -- /watchlist
│
└── 详情页 (StockDetailPage)     -- /stock/:code
    └── 从推荐/关注列表点击跳转
```

无侧边栏、无 Drawer、无层级嵌套导航。2 个 Tab + 1 个详情页，扁平结构。

### 页面列表

| # | 页面名 | 路由 | 关键状态变体 | 优先级 |
|---|--------|------|------------|-------|
| 1 | 推荐列表页 | /recommend | 默认(有数据)、加载中、加载失败(含缓存)、空状态、非交易日 | P0 |
| 2 | 关注列表页 | /watchlist | 默认(有数据)、搜索展开、搜索结果、空状态、左滑删除、右滑置顶 | P0 |
| 3 | 股票详情页 | /stock/:code | 默认(有数据)、加载中、新闻加载失败、评分计算中、未评分 | P0 |

## 核心组件规范

### 1. 股票列表项（StockListItem）

**用途**：推荐列表和关注列表中每只股票的展示单元。

**布局结构（ASCII 线框）**：
```
┌──────────────────────────────────────────┐
│  [评分Badge]   股票名称         ¥价格     │  12px 垂直
│  [波段低谷标签] 代码 SH  市值  +2.35%    │  12px 垂直
└──────────────────────────────────────────┘
 16px                                    16px
```

**7种状态**：

| 状态 | 视觉描述 | 触发条件 |
|------|---------|---------|
| 默认 | 白底，1px bottom border (#EEEEEE) | 正常展示 |
| 按下 | 背景 #F7F8FA | 用户触摸列表项 |
| 加载 | Skeleton 灰色占位 | 数据加载中 |
| 空状态 | 不适用于单项 | -- |
| 错误 | 单项无错误态 | -- |
| 置顶 | 左侧显示蓝色图钉图标 | 用户右滑置顶 |
| 预警 | 右侧价格旁显示红色铃铛图标 | 触发下跌预警 |

**尺寸**：全宽（减去 16px*2 水平内边距），高度约 64-72px

**实现要点**：
- Flutter: `ListTile` 自定义或 `Container` + `InkWell`
- 滑动操作：`flutter_slidable` 包或自实现 `Dismissible`
- 评分 Badge：28x20 圆角矩形（4px 圆角），居左
- 价格和涨跌幅右对齐，使用等宽数字字体

### 2. 评分 Badge（ScoreBadge）

**用途**：展示 1-10 评分的彩色标识。

**视觉规范**：

| 分数范围 | 背景色 | 文字色 | 尺寸 | 圆角 |
|---------|--------|--------|------|------|
| 8-10 | #E6432D | #FFFFFF | 28x20 | 4px |
| 5-7 | #D4A017 | #FFFFFF | 28x20 | 4px |
| 1-4 | #1DB954 | #FFFFFF | 28x20 | 4px |
| 计算中 | #EEEEEE | #8C8C8C | 56x20 | 4px |
| 数据不足 | #EEEEEE | #8C8C8C | 76x20 | 4px |

**文字**：
- 有评分：显示数字（如 "8"）
- 计算中：显示「...」
- 数据不足：显示「N/A」

### 3. 波段低谷标签（BandLowTag）

**用途**：标识当前处于波段低谷的股票。

**视觉**：橙黄色背景标签 `rgba(230, 147, 33, 0.12)`，文字色 `#E69321`，文字「波段低位」。
尺寸：自适应宽度，高度 18px，内边距 4px 8px，圆角 4px。
位置：紧跟在评分 Badge 右侧或股票名称下方。

### 4. 搜索框（SearchBar）

**用途**：关注列表顶部，搜索添加股票。

**布局**：
```
┌──────────────────────────────────────┐
│  [搜索图标]  搜索股票代码或名称    [X] │
└──────────────────────────────────────┘
```

**状态**：

| 状态 | 视觉描述 | 触发条件 |
|------|---------|---------|
| 默认 | 灰色边框 #E0E0E0，浅灰占位文字 | 未聚焦 |
| 聚焦 | 蓝色边框 #1A6AFF，轻微阴影 | 用户点击 |
| 输入中 | 蓝色边框，显示输入文字 | 输入字符 >= 1 |
| 搜索结果 | 搜索框下方展开结果列表 | 输入后匹配 |
| 无结果 | 结果列表中显示「未找到匹配股票」 | 无匹配 |

**尺寸**：全宽（减去 32px 水平内边距），高度 40px，圆角 8px。

### 5. Toast 通知

**用途**：操作反馈（添加关注、删除、置顶等）。

**视觉**：
- 位置：屏幕底部，距底部 Tab 栏上方 16px
- 背景：#333333（深灰），文字 #FFFFFF
- 高度：40px，圆角 8px，水平内边距 16px
- 持续：成功 2s，失败 3s

**实现要点**：
- Flutter: `Overlay` 或 `ScaffoldMessenger.showSnackBar`
- 不遮挡列表内容

### 6. 离线/缓存 Banner（CacheBanner）

**用途**：网络异常时提示用户当前展示的是缓存数据。

**视觉**：
- 位置：列表顶部，紧贴搜索框/页面标题下方
- 背景：#FFF8E1（浅黄），左侧黄色图标
- 文字：「离线模式，数据更新于 14:30」或「数据更新失败，显示缓存数据」
- 高度：32px，全宽
- 可关闭（右侧 X 按钮）

### 7. 空状态（EmptyState）

**用途**：列表无数据时的引导页面。

**三件套**：
- 图标：64x64 灰色图标（#BDBDBD）
- 标题：15px，#333333
- 副标题：13px，#8C8C8C
- CTA 按钮（可选）：品牌色背景，白色文字

| # | 模块/功能 | 描述 | 对应 PRD | 状态变体 |
|---|----------|------|---------|---------|
| 1 | 页面标题栏 | 显示「推荐」+ 日期 + 副标题 | F001 | 默认 |

**(a) 默认状态（有数据）**：
- 标题栏：左对齐「推荐」（h1 20px 600），右侧日期（caption 11px #8C8C8C）
- 副标题：「今日全市场波段分析结果」（body 13px #8C8C8C）
- 分组标题：左对齐「短线波段」（h3 15px 500），右侧数量（caption 11px #8C8C8C）
- 列表项：按评分排序（高到低），每组内部按评分降序

**(b) 加载状态**：
- 标题栏正常显示
- 分组标题区域显示灰色 Skeleton 块（宽 80px 高 16px）
- 列表项显示 6-8 个 Skeleton 卡片：
  - 第一行：矩形块（32px 宽，12px 高）+ 矩形块（80px 宽）+ 矩形块（50px 宽，右对齐）
  - 第二行：矩形块（100px 宽）+ 矩形块（60px 宽，右对齐）
  - Skeleton 颜色：#F0F1F3，闪烁动画

**(c) 加载失败（有缓存）**：
- 列表正常显示缓存数据
- 列表顶部显示黄色 Banner：「数据更新失败，显示缓存数据」，右侧 X 关闭
- Banner 下方小字：「数据更新于 14:30」（caption 11px #8C8C8C）

**(d) 加载失败（无缓存）**：
